Arquitectura General del Sistema
Sistema multi-tenant Next.js 15 con bases de datos aisladas por tenant en Neon PostgreSQL. Cada entidad pública opera en su propio dominio con datos completamente separados.

Capas del Sistema

🌐

Edge Runtime (Vercel)

Middleware multi-tenant · Resolución de dominio → tenant · SuperAdmin guard · Headers de seguridad

🔀

Next.js App Router

Route Handlers (API) · Server Components · Server Actions · Layouts anidados (Public / Admin / SuperAdmin)

⚙️

Capa de Servicios (lib/)

tenant.ts · auth · validations (Zod) · mail (Resend) · upload · storage · rate-limit · superadmin-auth

🔐

Autenticación

NextAuth v5 (JWT 24h) · SuperAdmin JWT separado (jose, 8h) · Recuperación de contraseña con tokens

🗄️

Base de Datos

Meta-DB (tenants, superadmins) · Tenant-DB aislada por tenant (50+ modelos Prisma) · Neon PostgreSQL serverless
